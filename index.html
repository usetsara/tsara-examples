Notes

  • Replace placeholder keys and references before testing.
  • Stablecoin status refresh examples use provider_reference plus refresh=1.
  • Bill crypto support flow uses fetch, reconcile, and retry as separate support operations.
  • Payout flows are asynchronous; create returns a local pending state and later reads show processing or final status.
  • Payout and refund webhooks use the same X_TSARA_SIGNATURE signature header as checkout webhooks.
